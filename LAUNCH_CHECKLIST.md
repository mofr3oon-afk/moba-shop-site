# MOBA SHOP - Final Launch Checklist

استخدم القائمة دي قبل النشر النهائي، ويفضل اختبارها مرة على الكمبيوتر ومرة على موبايل حقيقي.

## أول فتح وسرعة
- [ ] افتح الموقع بعد مسح الكاش وتأكد أن Skeleton يظهر بسرعة ثم الداتا الحقيقية.
- [ ] تأكد أن مفيش بيانات قديمة أو منتجات وهمية قبل تحميل الكتالوج.
- [ ] افتح الموقع من موبايل وتأكد أن الهيدر، العروض، الألعاب، وفرعون ظاهرين بتناسق.

## الطلب والدفع
- [ ] اعمل طلب InstaPay كامل.
- [ ] اعمل طلب Wallet كامل.
- [ ] جرب رفع صورة صحيحة JPG أو PNG أو WEBP.
- [ ] جرب رفع ملف غلط وتأكد أن الرسالة مفهومة للعميل.
- [ ] اضغط تنفيذ الطلب مرتين بسرعة وتأكد أن الطلب لا يتكرر.
- [ ] راجع Modal التأكيد قبل إرسال الطلب: المنتجات، الإجمالي، الدفع، الموبايل، و ID.
- [ ] تأكد أن الأخطاء تظهر تحت الحقل نفسه عند نقص رقم الموبايل أو الصورة أو آخر 3 أرقام.

## حالات الموقع
- [ ] حالة Available تسمح بالطلب.
- [ ] حالة Busy تسمح بالطلب وتعرض رسالة ضغط.
- [ ] حالة Closed تسمح بالطلب وتوضح أن التنفيذ عند العودة.
- [ ] حالة Maintenance تمنع الطلب من الواجهة ومن API الطلبات.

## الصفحات العامة
- [ ] افتح لعبة PUBG.
- [ ] افتح عرض من الصفحة الرئيسية.
- [ ] افتح رابط منتج مباشر.
- [ ] افتح الشروط والسياسات.
- [ ] افتح FAQ.
- [ ] افتح رابط غير موجود وتأكد من صفحة 404.
- [ ] ابعت رابط الموقع على واتساب أو تليجرام وتأكد من Social Preview.

## متابعة الطلب
- [ ] تابع طلب برقم الموبايل.
- [ ] افتح تفاصيل طلب وتأكد من التقسيم: بيانات الطلب، المنتجات، الدفع، الحالة، الدعم.
- [ ] ارفض طلب من الأدمن وتأكد أن السبب يظهر للعميل.
- [ ] حوّل طلب إلى مكتمل وتأكد أن الحالة تظهر صح.

## لوحة الأدمن
- [ ] افتح الأدمن من الكمبيوتر.
- [ ] افتح الأدمن من موبايل.
- [ ] جرب البحث برقم الطلب ورقم الموبايل و ID.
- [ ] جرب فلتر اللعبة وطريقة الدفع والحالة.
- [ ] جرب Export CSV.
- [ ] جرب Health Check.
- [ ] راجع Error Logs.
- [ ] تأكد أن طلبات TEST تظهر Badge TEST ولا تدخل في إجمالي المبيعات أو التقارير أو Export المبيعات.

## PWA و Cache
- [ ] جرب Add to Home Screen.
- [ ] بعد رفع نسخة جديدة، افتح الموقع وتأكد أن النسخة الجديدة تظهر بدون داتا قديمة.
- [ ] تأكد أن Service Worker لا يكاش API أو admin أو catalog أو الطلبات.

## Telegram و Supabase
- [ ] تأكد أن الطلب وصل Supabase.
- [ ] تأكد أن رسالة الطلب وصلت Telegram.
- [ ] لو Telegram فشل، تأكد أن الخطأ يظهر في الأدمن فقط بدون كشف أسرار للعميل.
