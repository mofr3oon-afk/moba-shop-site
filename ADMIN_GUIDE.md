# MOBA SHOP Admin Guide

## تغيير حالة الموقع

افتح لوحة الأدمن ثم صفحة الإعدادات أو الأمان، واختر حالة الموقع:

- Available: الطلبات تعمل بشكل طبيعي.
- Busy: الطلبات تعمل مع رسالة ضغط.
- Closed: الطلبات تعمل مع رسالة خارج مواعيد التنفيذ.
- Maintenance: الطلبات تتوقف من الواجهة ومن API الطلب نفسه.

حالة الموقع Manual فقط ولا تتغير تلقائيا.

## قبول أو رفض طلب

افتح الطلب من صفحة الطلبات:

- استلام: لتحديد المسؤول عن الطلب.
- جاري التنفيذ: عند بدء تنفيذ الشحن.
- تم الشحن: عند اكتمال الطلب.
- يحتاج تعديل: لو فيه مشكلة في ID أو الصورة أو المبلغ.
- رفض/إلغاء: عند عدم إمكانية تنفيذ الطلب.

اكتب سبب واضح في خانة الملاحظة قبل إرسال حالة مشكلة أو رفض.

## تعديل بيانات طلب

داخل تفاصيل الطلب افتح "تعديل بيانات الطلب".

المسموح تعديله:

- ID
- اسم اللاعب
- رقم العميل
- ملاحظة داخلية

لا يمكن تعديل السعر من هنا. أي تعديل يتسجل في Audit Log.

## Test Order

من صفحة الطلبات اضغط "فتح طلب TEST".

استخدمه لتجربة الشحن كامل بدون إدخاله في المبيعات.

لحذف كل طلبات TEST اضغط "حذف طلبات TEST" واكتب DELETE.

## Export

من صفحة الطلبات اضغط Export CSV لتصدير الطلبات الظاهرة حسب الفلاتر الحالية.

## Health Check

افتح Health Check للتأكد من:

- Supabase reachable
- Telegram reachable أو not configured
- Storage/upload ready
- Payment methods configured
- Required ENV missing

لا تظهر أي Tokens أو Secrets في الصفحة.

## فشل Telegram

لو Telegram فشل، يظهر داخل الطلب أن الحالة فشل/معلق، وتظهر التفاصيل العامة في Health Check أو Error Logs بدون كشف أسرار.

## حماية الحذف

أي حذف كبير مثل حذف طلبات TEST يحتاج كتابة DELETE.
